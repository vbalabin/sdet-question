решение докер-в-докере принято на основе
https://www.jenkins.io/doc/tutorials/build-a-python-app-with-pyinstaller

docker-compose поднимает окружение
селеноид использует докер-в-докере, который поднимается чистым без images
когда окружение поднимется нужно вручную зайти в контейнер докера 
и выписать pull тех браузеров, которые прописаны browsers.json

Dockerfile в папке project билдим сами в image
рядом лежит скрипт, который интерактивно этот image поднимает
